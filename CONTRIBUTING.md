# Contribution guidelines

## For Issues

0. Check for **similar issues** first!
1. Provide as **many details** as possible

## For Pull Requests

0. Indent using **2 spaces**
1. Use **double quotes** for strings

- Exception: `'"'`

1. Write only **one expression per line**

- Bad: `foo(); bar();`

0. Write **valid JavaScript**, a code linter is recommended!
1. **Explain code** that is not obvious (comment above)
2. If you add new functions, **add documentation** (comment above)

- Wrap function arguments in `+` signs (e.g. `the +type+ is used forâ€¦`)

0. If you add new functions, **add tests** (test/test.js)
1. If you fixed an issue, **add a test** that would catch this issue
2. Run **tests** (`npm test`)
3. Make sure as much of your code as possible is **covered by tests**
4. Double check that your pull request is **awesome** :)
